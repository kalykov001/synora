Архитектурный подход

Как соединить React Native, Supabase и Gemini

A

Прямые вызовы из приложения

Gemini API вызывается напрямую из React Native. Supabase — только для auth и БД.

📱 App → Gemini API (прямо)
📱 App → Supabase (auth + данные)

+

  • Проще всего
  • Нет промежуточного слоя

−

  • API ключ хранится в приложении ⚠️
  • Нет rate limiting
B

Supabase Edge Functions (рекомендую)

AI-запросы идут через Supabase Edge Functions (Deno). Ключи хранятся на сервере.

📱 App → Supabase Edge Fn → Gemini API
📱 App → Supabase (auth + данные + PDF storage)

+

  • API ключ защищён
  • Можно добавить rate limiting
  • Один провайдер (Supabase)
  • Удобно позже добавить монетизацию

−

  • Чуть сложнее деплой
C

Отдельный Node.js бэкенд

Express/Fastify сервер на Railway/Render. Полный контроль над логикой.

📱 App → Node.js API → Gemini API
📱 App → Supabase (auth + данные)

+

  • Максимальный контроль
  • Любая логика

−

  • Нужен отдельный хостинг
  • Больше инфраструктуры
  • Дольше

Моя рекомендация: B — Edge Functions безопасны, просты и уже внутри Supabase
